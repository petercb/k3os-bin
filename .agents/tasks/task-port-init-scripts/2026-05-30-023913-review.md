## Finalizer wiring fix verification

The fix in commit `2919707` adds three fields to the `Finalizer` construction in `postChroot()`:

```go
fin := &finalize.Finalizer{
    FS:            fs,
    Mounter:       mounter,
    Cmd:           cmd,
    SleepFunc:     time.Sleep,
    CmdlineReader: readCmdline,
    RandFunc:      cryptoRandUint32,
    VirtDetector:  detectVirt,
}
```

Each implementation satisfies its consumer's type contract:

- `SetupTTYs` calls `f.CmdlineReader()` expecting `(string, error)` — `readCmdline` returns `/proc/cmdline` contents as a trimmed string.
- `SetupHostname` calls `f.RandFunc()` expecting `(uint32, error)` — `cryptoRandUint32` generates a cryptographically random uint32.
- `SetupServices` calls `f.VirtDetector()` expecting `([]string, error)` — `detectVirt` returns parsed `virt-what` output or `(nil, nil)` on failure.

The `detectVirt` non-fatal design matches the original shell where `virt-what` might not be installed and was guarded with `|| true`. The Go version suppresses the error and returns an empty slice, causing `SetupServices` to skip VM-specific symlinks.
