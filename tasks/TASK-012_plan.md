## Objective

Migrate from the deprecated `github.com/moby/moby/pkg/reexec` package to its
replacement `github.com/moby/sys/reexec`. The old package is unmaintained and
will eventually be removed. The new package has a different API contract that
must be carefully handled to avoid panics at registration time.

---

## Context

### Key API Difference

The new `github.com/moby/sys/reexec` package has a critical behavioral change:

1. **`Register(name, ...)`** panics if `name` contains a path component (i.e.,
   a `/` character). Only basenames are allowed.
2. **`Init()`** now matches on `filepath.Base(os.Args[0])` instead of the full
   `os.Args[0]` value.

This means the current registrations of `/init` and `/sbin/init` (which both
map to the same `initrd()` function) must be consolidated into a single
registration of `"init"`. Since `Init()` uses `filepath.Base(os.Args[0])`,
invoking the binary as `/init` or `/sbin/init` will both resolve to `"init"`
and trigger the same handler.

### Current State

| File | Import | Usage |
|------|--------|-------|
| `main.go` | `github.com/moby/moby/pkg/reexec` | `Register("/init", ...)`, `Register("/sbin/init", ...)`, `Register("enter-root", ...)`, `Init()` |
| `internal/enterchroot/enter.go` | `github.com/moby/moby/pkg/reexec` | `reexec.Self()` |

**Important**: `internal/modalias/modalias.go` still imports
`github.com/moby/moby/pkg/parsers/kernel`, so `github.com/moby/moby` cannot
be fully removed from `go.mod`. Only the `pkg/reexec` import paths change.

### Files to Modify

| File | Change |
|------|--------|
| `main.go` | Change import, consolidate `/init` + `/sbin/init` to `"init"` |
| `internal/enterchroot/enter.go` | Change import path |
| `go.mod` | Add `github.com/moby/sys/reexec` dependency |
| `go.sum` | Regenerated by `go mod tidy` |

### Files Unaffected

| File | Reason |

### Step 1: Write Contract Tests (TDD - RED Phase)

**What**: Write tests that validate the reexec contract under the new API rules
before making any import changes.

**Tests to write**:

1. **Basename registration test**: Verify that the registration name `"init"`
   (without path components) does not panic and is correctly matched when
   `os.Args[0]` is `/init`, `/sbin/init`, or just `init`.
2. **Self() path validity test**: Verify that `reexec.Self()` returns a valid
   path to the current executable (used in `enterchroot/enter.go` for finding
   embedded squashfs data).

**Files**:

- `internal/enterchroot/enter_test.go` (extend existing tests if appropriate)
- Possibly a new `reexec_test.go` in the root package or a test helper

**Verification**:

```bash
docker run --rm -v "$(pwd)":/app -w /app golang:1.24-bookworm go test -race -run TestReexec ./...
```

**Conventional commit**: `test: add contract tests for reexec basename registration`

---

### Step 2: Update Imports and Consolidate Registrations (TDD - GREEN Phase)

**What**: Replace the old import with the new one and fix the registration calls.

**Changes**:

1. In `main.go`:
   - Change import from `github.com/moby/moby/pkg/reexec` to
     `github.com/moby/sys/reexec`
   - Replace `reexec.Register("/init", initrd)` and
     `reexec.Register("/sbin/init", initrd)` with a single
     `reexec.Register("init", initrd)`
   - `reexec.Register("enter-root", enterchroot.Enter)` stays unchanged
     (already a basename)

2. In `internal/enterchroot/enter.go`:
   - Change import from `github.com/moby/moby/pkg/reexec` to
     `github.com/moby/sys/reexec`

3. Run `go get github.com/moby/sys/reexec` to add the dependency.

**Verification (TDD GREEN)**:

```bash
docker run --rm -v "$(pwd)":/app -w /app golang:1.24-bookworm go test -race ./...
docker run --rm -v "$(pwd)":/app -w /app golang:1.24-bookworm go build -o /dev/null .
```

**Conventional commit**: `refactor: migrate reexec import from moby/moby to moby/sys`
