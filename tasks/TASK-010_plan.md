## Objective

Upgrade the Go version from 1.21.9 to 1.24 (current latest stable) across `go.mod`, CircleCI configuration, and the devcontainer definition. Ensure all tests, linting, and builds continue to pass. Use TDD to verify the upgrade does not introduce regressions. The Go 1.24 toolchain introduces `go.sum` format changes, new vet checks, and loop variable semantics (from 1.22) that may surface new warnings.

### Key Changes in Go 1.22-1.24 Affecting This Codebase

1. **Loop variable semantics (Go 1.22)**: Each iteration of a `for` loop creates a new variable. This eliminates a class of closure bugs but may cause `vet` or `staticcheck` to flag previously-hidden issues.
2. **`go.sum` format**: Go 1.24 uses a condensed `go.sum` format; running `go mod tidy` will rewrite it.
3. **`go vet` improvements**: New checks added in 1.22-1.24 may flag existing code.
4. **`golangci-lint` compatibility**: The project uses golangci-lint v2 which supports Go 1.24.
5. **`toolchain` directive**: Go 1.21+ introduced the `toolchain` directive in `go.mod`. When upgrading to `go 1.24`, Go may auto-add a `toolchain go1.24.x` line.

## Open Questions for Review

1. **Target version**: This plan targets Go 1.24 (latest stable as of 2025). If a more conservative upgrade to 1.22 or 1.23 is preferred, the plan steps remain identical -- only the version numbers change.

2. **`toolchain` directive**: Go 1.21+ auto-manages the `toolchain` directive in `go.mod`. Should we explicitly set it (e.g., `toolchain go1.24.4`) or let `go mod tidy` manage it automatically?
